## Decisions made

- **Polymorphic `idea_tags` replaces per-entity join tables.** See ADR-005. Added to support tagging ideas to any entity type without a migration per type. Migrated CLIENT-01's `idea_projects` helpers to read/write `idea_tags`. Old table remains but is unused.
- **Entity types as application-level union, not Postgres enum.** Adding a new taggable type is one-line in TS, no migration.
- **IdeaCaptureModal handles autoTag itself** (create → tag) rather than exposing it via a separate server action wrapper. Keeps the modal as the single capture surface; callers just pass `autoTag`.
- **IdeasPanel owns the tags state** and passes `onTagsChange` down. This keeps tab counts live after tag changes without needing `router.refresh()`. IdeasList still manages its own internal state but mirrors up.
- **No template file for the IDEA-01 pattern.** The global capture + flat list combination is unlikely to recur as-is; the IdeasList and IdeasPanel molecules are the reusable artefacts.
- **Session's Sonner toast reuse.** The "Saved" visual indicator from InlineField (`debouncedSaveToast`) is now duplicated inline in `ideas-list.tsx`. Not yet extracted — if a third consumer appears, promote to a shared helper.

## Context for future sessions

- **`idea_projects` table is deprecated but still exists physically.** Drizzle schema still defines it; CLIENT-01's wrapper functions no longer reference it. Don't add new code that writes to it. Use `idea_tags` directly or via the existing `tagIdea` / `untagIdea` helpers.
- **Entity types that can be tagged** are defined in one place: `IdeaTagEntityType` in `lib/types/ideas.ts`. To add a new type, also add: (a) a query in `listTaggableEntities`, (b) an icon + label in `ENTITY_TYPE_ICONS` / `ENTITY_TYPE_LABELS` in `ideas-list.tsx`. That's it — no migration, no schema change.
- **IdeasPanel can drop into any entity detail page.** Pass `entityType`, `entityId`, `entityLabel`, plus the three data props (allIdeas, tagsMap, taggableEntities) from the page-level loader. Label should be human-readable: "this mission", "this offer", etc.
- **Tags are resolved by name in the page loader**, not in the query layer. The `entityName` on a tag is `null` by default; pages build a lookup Map from `listTaggableEntities()` and enrich before passing down. Keeps the query layer simple.
- **IdeaCaptureModal is mounted in two places:** the TopToolbar (global, no autoTag) and inside IdeasPanel (per-workspace, with autoTag). The TopToolbar one continues to capture context via `usePathname` regardless of where the user is.
- **Migrations 0015 (ideas) and 0018 (idea_tags) were applied during this session.** Migration 0017 (CLIENT-01) came from the previous session.
