## Decisions made

- **INP-01 updated, not rebuilt** — the original auto-extract spec was superseded by INP-11. Updated brief and skill rather than building to old spec then refactoring.
- **`pending_inputs` cleared, not migrated** — 9 legacy items were extracted with the old prompt. Cleared rather than migrating since the processing model changed entirely.
- **Analysis modes use Gemini Pro** — `gemini-3.1-pro-preview` for batch/reflective/synthesis. Anthropic credits need topping up; Gemini Pro is inexpensive and output quality is strong. Can switch back later.
- **`extractedText` reused for raw transcript storage** — no separate `raw_text` column. The existing field semantics work for Krisp transcripts (text extracted from Krisp = the transcript itself).
- **Analysis results don't write to graph yet** — batch/reflective/synthesis produce structured analysis documents stored in `processing_runs.analysis_result`. Graph commit mapping (insights → Idea nodes, themes → Concept nodes, etc.) is designed but not implemented. Individual extraction → graph commit works via the existing `commitExtraction()` flow.
- **"Save analysis" / "Dismiss" over "Mark reviewed" / "Skip"** — Ellie's feedback: the old labels didn't communicate what happened. New labels express intent clearly.
