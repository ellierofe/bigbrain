## Decisions made

- **`status` varchar instead of `is_active` boolean** — three-state lifecycle (`draft` / `active` / `archived`) needed for creation modal flow (Save as draft before INF-06 is live) and soft delete. Simple boolean couldn't represent draft state.
- **`useEffect` on `open` prop for archive modal trigger** — `Dialog.onOpenChange` only fires when the dialog initiates its own close (e.g. ESC, overlay click). Since the parent controls `open` directly via `setState`, `onOpenChange(true)` never fires. `useEffect` watching `open` is the correct pattern for this controlled-open modal approach. Documented in ADR-001.
- **`TooltipTrigger render={<span />}` when wrapping a Button** — avoids button-inside-button HTML nesting that causes React hydration failures. Documented in ADR-001.
- **`getSegmentDependents` is a stub returning `[]`** — dependent tables (offers, knowledge assets, content pillars) don't exist yet. Will extend when DNA-04+ are built.
- **BRAND_ID hardcoded** — single-user app, will be replaced with session lookup when INF-05 is live.
