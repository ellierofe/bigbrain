## What was done

- `02-app/app/(dashboard)/dna/audience-segments/[id]/segment-detail-view.tsx` — added `nativeButton={false}` to the cards-view Button that uses `render={<Link />}`. Base UI warns when `nativeButton` is true (default) and the rendered element isn't a native `<button>`.

**Layout: viewport-locked chrome, scrolling content panes:**

- `02-app/app/(dashboard)/layout.tsx` — `main` changed from `overflow-y-auto` to `overflow-hidden flex flex-col` — page no longer scrolls as a whole
- `02-app/app/(dashboard)/dna/audience-segments/[id]/segment-detail-view.tsx`:
  - Two-column layout: content pane changed from `overflow-y-auto` to `flex flex-col min-h-0`
  - `Tabs` root given `flex-1 min-h-0` so it fills the pane height
  - All three `TabsContent` panels given `overflow-y-auto min-h-0` — only tab content scrolls
  - Left aside: `sticky top-0 self-start` removed, replaced with `overflow-y-auto` — scrolls independently if fields overflow

**Result:** Nav sidebar, page header, segment switcher, and tab bar are all fixed. Only the left aside and the active tab's content panel scroll.

## Decisions made

- **Per-pane scroll rather than page scroll** — matches the legacy Webflow design (`.main_page_structure` is `100svh overflow:hidden`, `.scrollbox_content_wrap_main` scrolls independently). Every DNA detail view should follow this same pattern: fixed chrome, scrolling content zone(s).
- **`nativeButton={false}` required when `render` prop passes a non-button element** — Base UI `Button` defaults `nativeButton={true}`, which expects the rendered element to be a native `<button>`. Using `render={<Link />}` renders an `<a>` tag, triggering the warning. Set `nativeButton={false}` whenever `render` produces a non-button. Updated ADR-001.

## What came up that wasn't planned
